区间 DP

简介

与线性 dp 不同，区间 dp 以区间长度作为 dp 的阶段，用左端点和右端点来表示每个维度，决策一般就是划分区间的方法，一个大区间一般由几个比它小的区间得来，与线段树十分相似

282. 石子合并 - AcWing题库

我们可以用一个区间 [l,r][l,r][l,r] 来表示 l∼rl\sim rl∼r 堆石子合并需要的最少体力

存在一个整数 k(l≤k<r)k(l\leq k<r)k(l≤k<r) 在 l∼rl\sim rl∼r 这堆石子合并成一堆时由 l∼kl\sim kl∼k 和 k+1∼rk+1\sim rk+1∼r 这两堆石子合并而来

那么我们就可以以 kkk 来划分区间

F[l,r]=minl≤k<r(F[l,k],F[k+1,r]+∑i=lrA[i])F[l,r]=min_{l\leq k <r}({F[l,k],F[k+1,r]}+\sum_{i=l}^r A[i]) F[l,r]=minl≤k<r​(F[l,k],F[k+1,r]+i=l∑r​A[i])

那么对于 ∑i=lrA[i]\sum ^r_{i=l}A[i]∑i=lr​A[i] 我们可以使用前缀和来计算

然后我们枚举区间的长度，再枚举 lll 的位置，那么 rrr 的位置就可以由长度和 lll 的位置计算得出

1068. 环形石子合并 - AcWing题库

我们假设 nnn 个石子的合并是在两个点之间连一条边，由于合并 nnn 堆石子只需要 n−1n-1n−1 次合并，但是 nnn 个点的环却有 nnn 个空隙，所以我们连完所有边（合并完所有石子）后，一定会有 1 个空隙，所以就可以枚举这个空隙，把空隙的两边断开变成一条链，但是这种做法的复杂度为 O(n)O(n)O(n) 原本的石子合并问题的复杂度为 O(n3)O(n^3)O(n3)，所以总的复杂度就为 O(n4)O(n^4)O(n4)，不可取

一般的做法实际是把长度为 nnn 的环展开成一条链，然后再复制一遍形成一条长度为 2n2n2n 的链

如果在 1 和 2 处断开就会得到 2,3,...,n,12,3,...,n,12,3,...,n,1

如果在 2 和 3 处断开就会得到 3,4,...,n,1,23,4,...,n,1,23,4,...,n,1,2

所以就可以直接利用上一题的做法来做

那么这样的做法的时间复杂度就为 O((2n)3)=O(8n3)O((2n)^3)=O(8n^3)O((2n)3)=O(8n3)，而且请读者记住这种做法，基本上大部分的环形 DP 问题都是用这个方式解决

写法有两种：1. 迭代式 2. 记忆化

一般区间只有一维的时候，一般使用迭代式，如果维比较多的话，一般使用记忆化，在后续的问题中会使用记忆化

320. 能量项链 - AcWing题库

本题与环形石子合并有相似之处，但是要注意细节上的差别

首先来看状态表示：f[L,R]f[L,R]f[L,R]

  • 集合：所有将 [L,R][L,R][L,R] 合并在一个珠子的方式（此处的 L,RL,RL,R 是题目中给的数据的下标）
  • 属性：Max

我们由题目叙述可知，a[L]a[L]a[L] 实际上是第 LLL 颗珠子的头标记，a[R]a[R]a[R] 实际上是第 R−1R-1R−1 颗珠子的尾标记 （这里使用 aaa 数组表示题目给的数据的值）

那么状态转移方程也就是 f[L,R]=max(f[L,K]+f[K,R]+w[L]×w[K]×w[R])f[L,R]=max(f[L,K]+f[K,R]+w[L]\times w[K]\times w[R])f[L,R]=max(f[L,K]+f[K,R]+w[L]×w[K]×w[R])

接下来考虑环形的情况：

根据题目举的例子 (2,3)(3,5)(5,10)(10,2)(2,3)(3,5)(5,10)(10,2)(2,3)(3,5)(5,10)(10,2)

假设我们随意合并一下（权值不是最大），那么第一步合并 1 2 珠子，变成 (2,5)(5,10)(10,2)(2,5)(5,10)(10,2)(2,5)(5,10)(10,2)

第二步合并当前的 1 2 珠子，变成 (2,10)(10,2)(2,10)(10,2)(2,10)(10,2)

再次合并当前的 1 2 珠子，变成 (2,2)(2,2)(2,2)

所以此时我们也就能发现，必然有一颗珠子在合并完之后头尾标记是相同的，换句话说，在 nnn 个点的环中，存在一个点 xxx，（指定从 xxx 开始，所有点顺时针编号）头x+1=尾x,...,尾x+n−1=头x头_{x+1}=尾_x,...,尾_{x+n-1}=头_x头x+1​=尾x​,...,尾x+n−1​=头x​，而在一番操作之后 x+1∼x+n−1x+1\sim x+n-1x+1∼x+n−1 的珠子全部合并掉，那么局面就会变成原本的 xxx 和 x+1∼x+n−1x+1\sim x+n-1x+1∼x+n−1 的所有珠子的结合体，(头x,尾x)(头x+1,尾x+n−1)(头_x,尾_x)(头_{x+1},尾_{x+n-1})(头x​,尾x​)(头x+1​,尾x+n−1​) 再合并得到最终的珠子 (头x,尾x+n−1)(头_x,尾_{x+n-1})(头x​,尾x+n−1​) 请注意 x+n−1x+n-1x+n−1 是实际的珠子数量，并不是 aaa 数组中的元素个数，x+n−1x+n-1x+n−1 个珠子需要 x+nx+nx+n 个元素，而又因为 尾x+n−1=头x尾_{x+n-1}=头_x尾x+n−1​=头x​ 所以 a[x+n]a[x+n]a[x+n] 顺理成章的就是 a[x]a[x]a[x]，所以我们发现如果 aaa 数组的元素 =2n=2n=2n，那么我们就可以满足所有情况

注意此时区间长度的最大值变成了 n+1n+1n+1，这是因为想要算出答案的话需要 a[x]∼a[x+n]a[x]\sim a[x+n]a[x]∼a[x+n] 一共是 n+1n+1n+1 个元素，最后枚举点 xxx，取 max 便得到了最终答案

1069. 凸多边形的划分 - AcWing题库

我们划分一个多边形成为三部分：左部分 + 中间的三角形 + 右部分

我们可以发现左部分怎么划分的不影响右部分的权重，所以可以使用区间 DP 来进行划分

这里的区间指某一个多边形的起始点至终点（我们按顺序给每一个点标号，如果某两个点相连后可以构造出一个多边形（包括三角形），我们易知这两个点之间至少还存在一个点，而且这个点的序号一定小于起始点和终点）

那么我们就可以得到状态表示：f[L,R]f[L,R]f[L,R]

  • 集合：表示由 L,L+1,...RL,L+1,...RL,L+1,...R 构成的多边形的方案

  • 属性：Min

状态计算：

枚举把这个多边形分为三部分的点 k(L<k<R)k(L<k<R)k(L<k<R)（点数 ≥4\geq4≥4 的时候很好理解，解释一下三角形的情况，由于 kkk 与 L,RL,RL,R 都是相邻的，而长度最少从 3 开始枚举，所以值并没有被更新过，是 0，加上不影响结果），那么左部分的值就为 f[L,k]f[L,k]f[L,k]，右部分的值就为 f[k,R]f[k,R]f[k,R]，再加上中间三角形的权值就行了

状态转移方程就为 f[L,R]=min(f[L,k]+f[k,R]+w[L]×w[k]×w[R])f[L,R]=min(f[L,k]+f[k,R]+w[L]\times w[k]\times w[R])f[L,R]=min(f[L,k]+f[k,R]+w[L]×w[k]×w[R])

479. 加分二叉树 - AcWing题库

二叉树的中序遍历：

这是一个递归的过程，遍历顺序为左根右

首先看当前节点有没有左节点，有就转到那个点上继续看，直到没有为止，然后输出当前点，再看这个点有没有右节点，如果有就转到那个点上再看重复以上过程，如果没有右节点那就回溯到自己的根节点

在这道题目中可以利用的性质：只有遍历完一个节点的左子树再输出这个节点再遍历这个节点的右子树

那么因为题目告诉我们中序遍历的序列为 1∼n1\sim n1∼n，所以这个对于任何一个子树，序列中的数都是连续的，那么我们就可以枚举根节点

对于 len=1len=1len=1 的情况，不枚举根节点

需要注意一下，否则理解会有偏差，右节点在根节点之后被访问，左节点在根节点之前被访问

状态表示 f[L,R]f[L,R]f[L,R]

  • 集合：所有中序遍历是 [L,R][L,R][L,R] 这一段的二叉树的集合
  • 属性：Max

状态计算：

枚举根节点 k(l≤k≤r)k(l\leq k\leq r)k(l≤k≤r) f[l][r]=max(f[l][k−1]×f[k+1][r]+w[k])f[l][r]=max(f[l][k-1]\times f[k + 1][r] + w[k])f[l][r]=max(f[l][k−1]×f[k+1][r]+w[k])（如果存在左子树右子树，如果有一个没有那么需要变成 1）

最后使用一个数组来记录 [L,R][L,R][L,R] 的根节点然后再用 DFS 求先序遍历就行了
