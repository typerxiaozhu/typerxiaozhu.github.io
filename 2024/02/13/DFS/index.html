深度优先搜索 DFS

DFS 的搜索顺序：尽量往深处搜，不走到头不回去，而在回去的途中又会看能不能往前走

数据结构：stack（调用系统栈）注意递归层数多的话可能会爆栈，为了避免爆栈可改成递推

空间复杂度：O(n)O(n)O(n)

关于是否需要恢复现场的问题：

当我们求解的问题是一个棋盘中走来走去时，不需要恢复现场（每个点都只会被遍历一次），而如果求的是棋盘的状态变化，需要恢复现场，课参考 BFS 中走迷宫与八数码

解释一下为什么，例如你在一个棋盘里面走，你走到下一步的时候，上一步一直是存在的，所以不能恢复现场，而状态变化问题就不一样

1118. 分成互质组 - AcWing题库

本题跟最大团问题很像，但是由于难度的原因这里不介绍

我们考虑每个数的决策，直观地来看可以分成两种：

  • 加进某一组中
  • 新开一组加进去

但是第一个决策我们可以优化成每次枚举一组中所有需要放的数，然后再枚举下一组，于是就可以变成“加进最后一组中” ，那么我们再考虑一下如果一个数可以放进最后一组还是否需要新开一组吗？答案是否定的，请读者记住，如果一个数新开一个组，那么之前的那个组是不会再放数字的，所以数字放在最后一组不会对答案产生任何影响。

每个数字放在一个组里面的顺序是无所谓的，所以我们不能使用排列的方式来搜索，必须用组合的方式搜索，我们可以按照下标从小到大的顺序进行枚举，传一个参数 start 记录上一个数的下标

同样的如果组里面的元素一样，顺序也是无所谓的，所以我们只需要加上 if (gc == 0) return; 就行了，解释一下，如果 gc == 0 表示我们当前回溯的点所在的组里面只有一个数（如果不止一个数的话会先回溯到在它后面的一个数，那样 gc 应该 = 1），由于回溯过程中只可以用在回溯到之前这个数之前所有已经回溯的数。所以假如有一组里面只有 1 个数，下一组里面有 xxx 个数，那么由于我们是每组每组枚举的，所以下一组里面的数的下标一定 > 当前这组里面的数的下标，所以说当前组里的数就一定可以被替换成下一组中的某一个数，假设被替换成下一组中的第一个数，那么按照顺序递归下去，当前组就会变成下一组，下一组就会变成当前组，对答案没有影响（组的顺序没有用，与数字同理），所以要剪掉

剪枝

之前提到，所有的 DFS 都可以对应一棵搜索树，但是在我们搜索的过程中，我们可能会发现某一枝是无用的，我们就可以不再往下递归，提高算法执行效率

常用的剪枝方法：

  • 优化搜索顺序 在搜索过程中，顺序并不是固定的，大部分情况下我们应该优先搜索分支较少的节点，在没有其他剪枝配合的情况下并没有用，搜到的节点数始终一样，但是如果有的话，我们就可能可以更早的发现这个枝不合法，提前剪掉
  • 排除等效冗余 比如我们只需要搜组合（选东西的时候不考虑顺序）
  • 可行性剪枝 当前方案不合法，剪掉
  • 最优性剪枝 当前方案的值已经比之前搜到过的解大了，无论如何不可能，剪掉
  • 记忆化搜索（DP）纯剪枝问题用的较少

167. 木棒 - AcWing题库

我们从小到大枚举原始木棒的长度 lenlenlen，当然 lenlenlen 必须是所有木棍的长度之和的约数，然后再枚举用哪些木棍拼成这个木棒

接着我们考虑剪枝：

  • 优化搜索顺序 按照长度从大到小枚举木棒

  • 排除等效冗余

    • 木棒拼接顺序无所谓，所以使用组合数方式枚举，参考 分成互质组 一题

    • 如果当前木棍加到当前木棒失败了，则直接略过后面所有长度相等的木棒

      使用反证法：假设我们有两个长度相等的木棍 3 号木棍 和 4 号木棍

      3 号加到当前木棒失败了，那么它就必须加到后续的某个木棒中（由于不考虑顺序所以不可能加到 4 号后面），那么假如 4 号可以加到当前木棒，那么就一定交换 3 号木棒 和 4 号木棒，矛盾，所以成立

    • 如果是木棒的第一根木棍失败了则整个方案失败，直接回溯

      由于我们不考虑顺序，所以放在最前面失败，放在哪里都会失败

    • 最后一根与第一根同理。为什么中间的木棍就不是？中间的木棍可以由长的木棍的一半和下一根木棍的前一半拼成，和第一根最后一根不一样

迭代加深

DFS 的搜索顺序是选择一个枝不断深入一直搜到底再回溯，假如我们一开始就选错了枝，答案又在教为浅的地方，那么就会白搜很多点。我们可以限制一个深度，每次每个枝只搜 1∼dep1\sim dep1∼dep 层，虽然这样我们会反复搜一些点，不过跟搜深层子树要搜的点的数量比还是少多了

所以当搜索树随着搜索的层数增长地非常迅速的时候，并且答案在较浅的层数里时，我们就应该使用迭代加深
